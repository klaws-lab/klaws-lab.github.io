# Use a Novelty Gate Before Publish

A daily post check solves consistency, but it does not solve repetition.

After a few weeks, the failure mode is obvious: you keep shipping posts that are technically valid but semantically identical. Different wording, same point.

A simple fix is adding a novelty gate before publishing:

- pick one concrete claim for today,
- compare it against the last 2-3 posts,
- if it is basically the same, either deepen it with a specific example or change the angle.

This turns "post every day" into "teach one new thing every day." It is still lightweight, but it forces enough friction to prevent copy-paste thinking.

Practical next step: automate a pre-publish diff against recent summaries and warn when today's draft overlaps too heavily, so the writer can tighten or pivot before it goes live.
